Implementation Guidelines

Color Requirements:

  • Primary buttons must have #3E213A background with white text
  • Accent buttons must have #FFD204 background with #3E213A text
  • Hover states should maintain text colors while reducing opacity to 85%

Chakra UI Implementation:

// Basic button usage
<Button variant="primary">Primary Button</Button>
<Button variant="accent">Accent Button</Button>

// With icon
<Button 
  variant="primary"
  leftIcon={<Icon />}
  size="md"
>
  Button with Icon
</Button>

// Link button
<Button 
  as={RouterLink}
  to="/some-path"
  variant="primary"
>
  Link Button
</Button>

Important Styling Notes:

  • Always use the variant prop with either "primary" or "accent" values
  • Never override button text colors with inline styles
  • Do not use the sx prop to modify button styling directly
  • For button customizations, extend the theme definitions in theme.ts
  • In Chakra UI, always apply color="white" to primary variant buttons if necessary

Recommended Approach

For best results maintaining consistent button styling across the application:

  1. Define colors in theme.ts - Put all brand colors and button styles in the theme file
  2. Include high-specificity CSS - Add CSS with !important flags to a global style sheet
  3. Use simple component syntax - Keep button JSX minimal (<Button variant="primary">)
  4. Consider programmatic enforcement - Use a component like ButtonStyleFixer to enforce styling at runtime

Final Button Implementation Checklist:

  • ✅ Primary variant has #3E213A background and white text
  • ✅ Accent variant has #FFD204 background and #3E213A text
  • ✅ Hover states maintain text color and reduce opacity to 85%
  • ✅ Focus/active states maintain correct text color
  • ✅ Button text remains readable in all states

To enforce consistent button styling across the application, add these CSS rules to a global stylesheet:

.chakra-button[data-variant="primary"],
a.chakra-button[data-variant="primary"],
button.chakra-button[variant="primary"],
.chakra-button[variant="primary"],

Troubleshooting Button Styling

If you encounter issues with button styling, particularly with text colors:

  1. Remove inline sx props - Don't use sx={{ color: "white" }} directly on buttons
  2. Check component inheritance - Ensure parent components aren't overriding button styles
  3. Use direct styling only as a last resort - Only if absolutely necessary, apply color="white" directly
  4. Review theme configuration - Ensure theme.ts has proper button variant definitions

Common Issues:

  • Blue text on primary buttons - Usually caused by inline styling conflicts or theme inheritance issues
  • Inconsistent hover states - Add _hover definitions to the theme with !important flags
  • Focus state styling - Include _focus and _active states in theme definitions
  • Link buttons styling - Ensure as={RouterLink} buttons maintain correct styling
